== COOKED-FAST GAME DEV GUIDE ==
Unity 6000.3.16f1 | C# | Isometric Cafe Tycoon
Last updated: Lesson 6 complete

== ARCHITECTURE ==
- Abstract class pattern: SpawnableFood (abstract) -> CoffeeCup, Pizza
- Plateau system: universal stacking tray used everywhere (stations, player, drop zones, future: customers, tables, workers)
- Push/Pop pattern: Station.Pop() -> Plateau.Pop() -> FoodPosition.Pop() (top-down delegation chain)
- Ability system: player/workers have modular abilities (HoldFoodAbility, future: HoldDishAbility, PayAbility, NavigationAbility)
- PlayerDetector: OnTriggerStay routes to correct ability via TryGetComponent

== KEY CLASSES ==

SpawnableFood (abstract MonoBehaviour):
  - float CleanYOffsetOnPlateau (serialized, per-food stacking height)
  - future: dirty offset, dirty mesh swap

FoodPosition (MonoBehaviour):
  - bool IsEmpty (init in Awake, not Start - critical for runtime-created positions)
  - SpawnableFood food (stored reference)
  - Push(SpawnableFood): reparent + localPos=zero, set isEmpty=false
  - Pop(): return food, null ref, set isEmpty=true

Plateau (MonoBehaviour):
  - Transform foodPositionsParent (serialized)
  - int maxCapacity (serialized)
  - bool IsFull, bool IsEmpty (properties)
  - float positionsYOffset (set from food.CleanYOffsetOnPlateau)
  - Push(food): find first empty pos -> push -> rearrange -> check full -> create new pos or mark full
  - Pop(): get last full pos -> pop -> check empty -> return food
  - GetFirstEmptyFoodPosition(): loop 0->n
  - GetLastFullPosition(): loop n->0 (grab from top)
  - GetFirstFullPosition(): loop 0->n (empty check after pop)
  - RearrangeFoodPositions(food): set all positions to Vector3.up * i * offset
  - CreateNewFoodPosition(): new GameObject + AddComponent, parent to foodPositionsParent, calc position from bottom sibling

FoodSpawnerStation (MonoBehaviour):
  - SpawnableFood spawnableFoodPrefab (serialized)
  - Plateau plateau (serialized)
  - float spawnDelay (serialized), float spawnTimer
  - Update -> HandleSpawnTimer -> TrySpawnFood -> SpawnFood
  - SpawnFood(): Instantiate prefab as child of station, then plateau.Push
  - Pop(): delegates to plateau.Pop()

FoodDropZone (MonoBehaviour):
  - Plateau plateau (serialized)
  - IsFull -> plateau.IsFull
  - Push(food) -> plateau.Push(food)

PlayerDetector (MonoBehaviour, RequireComponent HoldFoodAbility):
  - caches HoldFoodAbility in Awake
  - OnTriggerStay: TryGetComponent for FoodSpawnerStation or FoodDropZone, routes to ability

HoldFoodAbility (MonoBehaviour):
  - Plateau plateau (serialized, player's plateau)
  - grab/drop timers with 0.1s delay
  - HandleFoodSpawnerStation: check full -> timer -> station.Pop -> enable plateau -> plateau.Push
  - HandleFoodDropZone: check plateau active -> check dropzone full -> timer -> plateau.Pop -> dropZone.Push -> if empty disable plateau

PlayerAnimator:
  - switches between Idle/Walk and IdleWithPlateau/WalkWithPlateau based on plateau.activeInHierarchy

Customer (MonoBehaviour):
  - empty for now, will hold customer logic (navigation, ordering, sitting)

CustomerAnimator (MonoBehaviour):
  - same as PlayerAnimator (copied), will add sitting animation later
  - Animator animator, GameObject plateau (serialized)
  - ManageAnimations(moveVector, moveSpeed)

CustomerManager (Singleton MonoBehaviour):
  - static Instance (set in Awake)
  - Customer customerPrefab (serialized)
  - Pop(Vector3 spawnPosition): Instantiate customer at position, child of manager

FoodServingStation (MonoBehaviour):
  - empty for now, will handle serving food to customers

FoodServingCustomerManager (MonoBehaviour):
  - Transform spawnPoint (serialized)
  - int maxCustomers (serialized)
  - Queue<Customer> customers
  - Start -> InvokeRepeating SpawnNewCustomer every 1s
  - SpawnNewCustomer: check count < max -> CustomerManager.Instance.Pop -> enqueue

== PATTERNS & GOTCHAS ==
- Awake vs Start: FoodPosition uses Awake for isEmpty init (runtime AddComponent needs immediate init)
- Plateau starts disabled on player, enabled on first food grab, disabled when empty
- All models use single "palette" material with UV color selection
- Trigger colliders need: one side has Collider+Rigidbody(or CharacterController), other has trigger Collider
- Prefab renderer localPosition must be zero (asset imports may have offset - always check)
- Timer pattern: if(timer < delay) { timer += deltaTime; return; } timer = 0; // proceed
- Start() sets grabFoodTimer = delay so player can grab immediately on game start
